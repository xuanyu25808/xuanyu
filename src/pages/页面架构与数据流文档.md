## 概述

本项目是一个基于 uni-app 的菜谱管理应用，包含三个核心页面：
- **菜单页面** (`menu/index.vue`) - 浏览和选择菜品
- **菜谱管理页面** (`person/index.vue`) - 创建分类、添加/编辑/删除菜品
- **已点菜品页面** (`recipe/index.vue`) - 查看和管理已点菜品

数据管理采用两个 Store：
- **menuStore.js** - 管理菜单数据（分类和菜品）
- **orderStore.js** - 管理已点菜品数据

## 页面展示逻辑

### 1. 菜单页面 (`menu/index.vue`)

#### 1.1 页面结构
- **左侧分类列表**：垂直滚动，显示所有菜品分类
  - 每个分类项包含：分类图标、分类名称
  - 支持点击切换分类，当前激活分类高亮显示
- **右侧菜品列表**：垂直滚动，显示当前分类下的所有菜品
  - 每个菜品项包含：菜品缩略图、菜品名称、操作按钮（添加/移除）
  - 支持点击菜品查看详情（弹出详情弹窗）

#### 1.2 核心展示逻辑

**分类联动机制**：
```javascript
// 左侧点击分类时，右侧自动滚动到对应分组
handleLeftItemClick(index) {
  // 计算目标分组的位置
  // 动态设置 scrollTop，使目标分组滚动到可视区顶部
}

// 右侧滚动时，左侧自动高亮对应分类
onRightScroll(e) {
  // 根据当前滚动位置计算当前可视的分组
  // 更新左侧激活索引
}
```

**菜品状态显示**：
- 通过 `isDishSelected(groupId, itemId)` 判断菜品是否已添加到点菜列表
- 已添加：显示减号图标（移除）
- 未添加：显示加号图标（添加）

**详情弹窗**：
- 点击菜品主区域触发 `selectDish(item)`
- 使用 `FoodDetailsl` 组件展示菜品详情（用料、步骤）
- 通过 `selectedDish` 控制弹窗显示/隐藏

#### 1.3 生命周期与数据加载

```javascript

  // 每次页面显示时：
  // 1. 从 menuStore 重新加载最新菜单数据

  // 3. 计算右侧分组位置，用于滚动联动
  this.$nextTick(() => {
    this.calcSectionTops()
  })
```

### 2. 菜谱管理页面 (`person/index.vue`)

#### 2.1 页面结构
- **顶部操作区**：创建新分类的输入框和按钮
- **分类列表**：展示所有分类及其下的菜品
  - 每个分类卡片包含：
    - 分类标题和菜品数量统计
    - "添加菜品"和"删除分类"按钮
    - 该分类下的菜品列表（每个菜品可编辑/删除）

#### 2.2 核心展示逻辑

**分类管理**：
- 创建分类：输入分类名称，点击"创建分类"按钮
- 删除分类：点击"删除分类"，弹出确认对话框

**菜品管理**：
- 添加菜品：点击"添加菜品"，弹出表单弹窗
- 编辑菜品：点击菜品项的"编辑"按钮，弹出表单弹窗（预填充数据）
- 删除菜品：点击"删除"按钮，弹出确认对话框

**表单弹窗**：
- 包含字段：菜品图片（最多3张）、菜名、用料列表、步骤列表
- 支持动态添加/删除用料和步骤
- 表单验证：菜名为必填项

#### 2.3 数据持久化


## 数据传递机制

### 1. 页面间数据传递

#### 1.1 通过 Store 传递（主要方式）

**菜单数据传递**：
```
person页面（菜谱管理）
  ↓ saveMenuListToStorage()
menuStore（本地存储）
  ↓ getMenuList()
menu页面（菜单浏览）
```

**已点菜品数据传递**：
```
menu页面（添加菜品）
  ↓ saveDishesList()
orderStore（本地存储）
  ↓ getDishesList()
recipe页面（已点菜品）
```

#### 1.2 页面跳转传递

**Tab 切换**：
- `recipe` 页面 → `menu` 页面：使用 `uni.switchTab({ url: '/pages/menu/index' })`
- 通过 TabBar 切换，数据通过 `onShow` 生命周期重新加载

**组件通信**：
- 父组件 → 子组件：通过 `props` 传递数据（如 `FoodDetailsl` 组件接收 `dish`）
- 子组件 → 父组件：通过 `$emit` 触发事件（如 `FoodDetailsl` 触发 `close` 事件）

## Store 运用详解

### 1. menuStore.js

#### 1.1 核心功能

**数据存储**：
- 存储键：`menu_categories`
- 存储方式：使用 `uni.setStorageSync` 同步存储到本地

**主要方法**：

```javascript
// 加载菜单数据
getMenuList()
// 逻辑：
// 1. 尝试从本地存储读取数据
// 2. 如果本地有数据且长度 > 0，返回本地数据
// 3. 否则返回默认 mock 数据（深拷贝）

// 保存菜单数据
saveMenuListToStorage(menu)
// 逻辑：
// 1. 验证数据为数组格式
// 2. 序列化为 JSON 字符串
// 3. 存储到本地

// 恢复默认菜单
resetMenu()
// 逻辑：
// 1. 删除本地存储
// 2. 返回默认 mock 数据
```

#### 1.2 数据初始化策略

- **首次使用**：从 `mock/menu.js` 加载默认数据
- **后续使用**：优先使用本地存储的数据
- **数据回退**：如果本地数据解析失败，回退到默认数据

#### 1.3 使用场景

- **person 页面**：
  - `onLoad`：加载菜单数据用于展示
  - 创建/编辑/删除分类或菜品后：调用 `saveMenuListToStorage` 保存
- **menu 页面**：
  - `onShow`：调用 `getMenuList` 获取最新菜单数据

---

### 2. orderStore.js

#### 2.1 核心功能

**数据存储**：
- 存储键：`xuanyu_order_dishes_v1`
- 存储方式：使用 `uni.setStorageSync` 同步存储到本地

**主要方法**：

```javascript
// 加载已点菜品
getDishesList()
// 逻辑：
// 1. 从本地存储读取数据
// 2. 解析 JSON 字符串
// 3. 同步 TabBar 角标（显示菜品数量）
// 4. 返回菜品列表

// 保存已点菜品
saveDishesList(dishes)
// 逻辑：
// 1. 验证数据为数组格式
// 2. 序列化为 JSON 字符串
// 3. 存储到本地
// 4. 同步 TabBar 角标

// 清空已点菜品
clearDishesList()
// 逻辑：
// 1. 删除本地存储
// 2. 清除 TabBar 角标

// 同步 TabBar 角标
syncTabBadge(count)
// 逻辑：
// 1. 如果数量 > 0，设置角标文本
// 2. 否则移除角标
```

#### 2.2 TabBar 角标同步

- 每次加载或保存已点菜品时，自动更新 TabBar 角标
- 角标显示在第二个 Tab（index: 1），显示已点菜品数量

#### 2.3 使用场景

- **menu 页面**：
  - `onShow`：调用 `getDishesList` 构建 `orderedMap`，用于判断菜品是否已添加
  - 添加/移除菜品：调用 `saveDishesList` 保存
- **recipe 页面**：
  - `onShow`：调用 `getDishesList` 获取已点菜品列表
  - 删除菜品：调用 `saveDishesList` 保存
  - 清空菜品：调用 `clearDishesList` 清空

## 数据实时更新机制

### 1. 更新触发时机

#### 1.1 菜单数据更新

**触发场景**：
- 在 `person` 页面创建/编辑/删除分类或菜品
- 调用 `saveMenuListToStorage(this.menu)` 保存数据

**更新传播**：
```
person页面 saveMenuListToStorage()
  ↓
menuStore 本地存储更新
  ↓
menu页面 onShow() → getMenuList()
  ↓
页面数据自动更新（Vue 响应式）
```

**关键点**：
- `person` 页面修改数据后立即保存
- `menu` 页面在 `onShow` 时重新加载，确保数据同步

#### 1.2 已点菜品数据更新

**触发场景**：
- 在 `menu` 页面添加/移除菜品
- 在 `recipe` 页面删除/清空菜品
- 调用 `saveDishesList(list)` 或 `clearDishesList()` 保存数据

**更新传播**：
```
menu/recipe页面 saveDishesList()
  ↓
orderStore 本地存储更新
  ↓
TabBar 角标自动更新（在 saveDishesList 内部）
  ↓
menu页面 onShow() → getDishesList() → orderedMap 更新
recipe页面 onShow() → getDishesList() → dishes 更新
```

**关键点**：
- 数据修改后立即保存到本地存储
- TabBar 角标在保存时自动同步
- 页面切换时通过 `onShow` 重新加载数据

### 2. 响应式更新机制

#### 2.1 Vue 响应式系统

**数据绑定**：
```javascript
// menu 页面
data() {
  return {
    leftInfo: {
      menuList: [],  // Vue 响应式数据
      activeIndex: 0
    },
    orderedMap: {}  // Vue 响应式数据
  }
}

// 当 getMenuList() 返回新数据时
this.leftInfo.menuList = getMenuList()  // 触发视图更新
```

**数组更新**：
```javascript
// person 页面
this.menu.push(newCategory)  // Vue 检测到数组变化，自动更新视图
saveMenuListToStorage(this.menu)  // 保存到本地存储
```

#### 2.2 页面生命周期更新

**onShow 钩子**：
- 每次页面显示时触发
- 重新从 Store 加载最新数据
- 确保数据与本地存储同步

**示例**：
```javascript
// menu 页面
onShow() {
  // 重新加载菜单数据
  this.leftInfo.menuList = getMenuList()
  // 重新加载已点菜品数据
  this.loadOrdered()
  // 重新计算滚动位置
  this.$nextTick(() => {
    this.calcSectionTops()
  })
}
```

### 3. 数据一致性保证

#### 3.1 单一数据源

- **菜单数据**：`menuStore` 是唯一数据源
- **已点菜品数据**：`orderStore` 是唯一数据源
- 所有页面都从 Store 读取数据，不维护独立副本

#### 3.2 数据同步策略

**写入时同步**：
- 修改数据后立即调用 `saveMenuListToStorage` 或 `saveDishesList`
- 确保本地存储始终是最新状态

**读取时同步**：
- 页面显示时（`onShow`）重新加载数据
- 确保页面数据与本地存储一致

**TabBar 角标同步**：
- 在 `orderStore` 的 `getDishesList` 和 `saveDishesList` 中自动同步
- 确保角标始终显示正确的数量

### 3. 完整数据交互流程

```
用户操作流程：

1. 在 person 页面创建分类/菜品
   → saveMenuListToStorage() → menuStore 更新

2. 切换到 menu 页面

   → onShow() → getDishesList() → orderedMap 更新
   → 菜品状态图标自动更新（已添加/未添加）
```

## 总结

### 核心设计理念

1. **单一数据源**：所有数据统一存储在 Store 中，避免数据不一致
2. **本地存储持久化**：使用 uni-app 的本地存储 API，数据持久化保存
3. **生命周期同步**：通过 `onShow` 钩子确保页面数据与 Store 同步
4. **响应式更新**：利用 Vue 的响应式系统，数据变化自动更新视图

### 数据更新保证

- ✅ 数据修改后立即保存到本地存储
- ✅ 页面显示时重新加载最新数据
- ✅ TabBar 角标自动同步更新
- ✅ 页面间数据通过 Store 共享，保证一致性

### 扩展性考虑

- Store 方法设计简洁，易于扩展新功能
- 数据结构清晰，便于后续添加字段
- 本地存储键名带版本号（`_v1`），便于数据迁移
